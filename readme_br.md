## Descrição

Nest é um framework para criar aplicações server-side em <a href="https://nodejs.org" target="_blank">Node.js</a> de forma eficiente e escalável. Ele utiliza JavaScript moderno, é construído com <a href="https://www.typescriptlang.org" target="_blank">TypeScript</a> (mantendo a compatibilidade com JavaScript puro) e combina elementos de POO (Programação Orientada a Objetos), PF (Programação Funcional) e PRF (Programação Funcional Reativa).

<p>Por baixo dos panos, Nest utiliza <a href="https://expressjs.com/" target="_blank">Express</a>, mas também oferece compatibilidade com uma ampla gama de outras bibliotecas, como <a href="https://github.com/fastify/fastify" target="_blank">Fastify</a>, permitindo o uso fácil da vasta quantidade de plugins de terceiros disponíveis.</p>

## Filosofia

<p>Nos últimos anos, graças ao Node.js, JavaScript se tornou a "lingua franca" da web, tanto para aplicações frontend quanto backend, dando origem a projetos incríveis como <a href="https://angular.io/" target="_blank">Angular</a>, <a href="https://github.com/facebook/react" target="_blank">React</a> e <a href="https://github.com/vuejs/vue" target="_blank">Vue</a>, que melhoram a produtividade dos desenvolvedores e permitem a construção de aplicações frontend rápidas, testáveis e flexíveis. No entanto, no server-side, embora existam muitas bibliotecas, ferramentas e auxiliares excelentes para o Node, nenhuma delas resolve efetivamente o principal problema: a arquitetura.</p> <p>O Nest tem como objetivo fornecer uma arquitetura de aplicação pronta para uso, permitindo a criação descomplicada de aplicações altamente testáveis, escaláveis, fracamente acopladas e de fácil manutenção. Sua arquitetura é fortemente inspirada no Angular.</p>

Para dúvidas e suporte, utilize o [canal oficial no Discord]((https://discord.gg/G7Qnnhy)). A lista de issues deste repositório é **exclusivamente** para relatórios de bugs e solicitações de funcionalidades.

## Issues

Antes de abrir uma issue, recomendamos fortemente que você leia a [Lista de Issues](https://github.com/nestjs/nest/blob/master/CONTRIBUTING.md#-submitting-an-issue). Isso garantirá que sua contribuição seja clara, relevante e dentro dos padrões aceitos.

## Consultoria

Com o suporte oficial, você pode obter ajuda especializada diretamente da equipe principal do Nest. Oferecemos suporte técnico dedicado, estratégias de migração, orientações sobre melhores práticas (e decisões de design), revisões de Pull Requests (PRs) e aumento de equipe. Saiba mais sobre nosso [suporte aqui](https://enterprise.nestjs.com).

## Suporte

Nest é um projeto open source licenciado sob a licença MIT. Seu crescimento contínuo é possível graças aos patrocinadores e ao apoio de uma comunidade incrível de contribuidores. Se você deseja fazer parte desse grupo de apoiadores e contribuir para o avanço do projeto, por favor, [saiba mais aqui](https://docs.nestjs.com/support).
